# Li-routi Frontend 코드 리뷰 스타일 가이드

## 언어

- 모든 코드 리뷰 코멘트, PR 요약, 이슈 관련 응답은 **한국어**로 작성한다.
- 코드 식별자(변수명, 함수명, 클래스명 등)나 로그/에러 메시지를 인용할 때를 제외하고 영어를 사용하지 않는다.

## 리뷰 코멘트 우선순위 태그

리뷰 코멘트를 작성할 때는 아래 우선순위 태그를 코멘트 앞에 붙인다.

- `[P1]`: 필수 수정 (버그, 로직 오류, 보안 이슈 등 merge 전 반드시 반영해야 하는 사항)
- `[P2]`: 권장 수정 (가독성, 컨벤션, 구조 개선 등)
- `[P3]`: 제안/의견 (사소한 의견, nit, 선택 사항)

## 아키텍처

- 이 프로젝트는 Gradle 멀티모듈 구조(`app`, `core/*`, `feature/*`)를 따른다.
- `core:domain`, `core:common:kotlin`은 순수 Kotlin 모듈이므로 Android SDK 의존성을 추가하는 변경은 지적한다.
- `feature:*` 모듈 간 직접 의존은 지적한다 (공유가 필요하면 `core:common:*` 또는 Navigation 추상화 사용을 권장).
